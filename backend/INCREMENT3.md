Скоуп: заполнять два листа шаблона — «Арсенал (Д)» и «Реф.сервис (Д)» — значениями из `MetricsService.daily()`. Остальные листы/направления — позже.

## Шаблон
- Исходник: `/Users/admin/Downloads/Отчёты АТЦ — Финал для автоматизации (1).xlsx`.
- Скопировать в проект: `backend/templates/report_template.xlsx` (self-contained; исходник не трогать).
- Заполнять КОПИЮ шаблона, не сам шаблон.

## Карта ячеек (листы «Арсенал (Д)» и «Реф.сервис (Д)» идентичны)
Значения ПЛАН — колонка **D**, ФАКТ — колонка **F**.

| Ячейка (D=ПЛАН / F=ФАКТ) | Показатель | Источник (metrics) |
|---|---|---|
| C3 | АТЦ (название) | report_name |
| C4 | Дата | day (формат ДД.ММ.ГГГГ) |
| D8 / F8 | Нормочасы за день — общая выработка | normhours.plan / .fact |
| D9 / F9 | Выработка на 1 мастера | output_per_master.plan / .fact |
| D10 / F10 | Закрыто ЗН за день | closed_orders.plan / .fact |
| D14 / F14 | Выручка за день | revenue.plan / .fact |
| D15 / F15 | Себестоимость ЗЧ | cost.plan / .fact |
| D19 / F19 | Наценка на ЗЧ (%) | markup_pct.plan / .fact |
| D23 | Незакрытые ЗН старше 1 дня (шт.) | unclosed_over_1day.fact |
| D24 | ЗН без движения > 2 дней (шт.) | пока «—» (нужен механизм снимков, отд. инкремент) |

**НЕ ЗАПИСЫВАТЬ (это формулы Excel):** D16/F16 (Маржа), D17/F17 (Маржинальность), D18/F18 (Средний чек), вся колонка H (%). Их оставить как есть — Excel посчитает сам.

Примечания по объединённым ячейкам: C3 (АТЦ), C4 (дата), D23/D24 — писать в левую верхнюю ячейку диапазона.

## Расширения в metrics/repositories (нужны для строк 19 и 23)
1. `OrderRepository.open_orders_older_than_1day(division_key, day)` → int:
   - `Document_ЗаказНаряд`, фильтр: `ПодразделениеКомпании_Key eq guid'..' and Состояние_Key ne guid'<Закрыт>' and Date lt datetime'<day-1 00:00>'`.
   - вернуть количество (можно select Ref_Key и len, либо $inlinecount).
   - ⚠️ ТЗ: «не в состоянии Закрыт». Исключаем только «Закрыт». (Возможное уточнение — Отказ/Архив — оставить как есть, пометить комментарием.)
2. `DailyMetrics` дополнить:
   - `markup_pct: Metric` («Наценка на ЗЧ, %») — ФАКТ = (выручка − себест) / себест × 100 при себест>0, иначе None; ПЛАН = план-показатель код **000000024** (значение как есть, %, БЕЗ деления на дни месяца — это процент, а не поток).
   - `unclosed_over_1day: Metric` — ФАКТ = число из репозитория, ПЛАН = None.
   - ⚠️ Формула ФАКТ наценки% помечена как черновая — вынести расчёт так, чтобы легко поменять. Флаг на подтверждение владельцем.
3. В `plan_mapping.py` уже есть коды 024/029/067/068/069/070 — переиспользовать.

## Что сгенерировать
```
backend/templates/report_template.xlsx   # копия исходника
src/excel_reporter.py                     # ExcelReporter.fill_daily(report_name, day, metrics, out_path)
tests/test_excel_smoke.py                 # заполнить Арсенал и Реф.Сервис за дату, сохранить в output/
```
- `output/` добавить в .gitignore.
- ExcelReporter: загрузить копию шаблона (openpyxl, keep_vba=False), найти лист по report_name («Арсенал»→«Арсенал (Д)», «Реф. Сервис»→«Реф.сервис (Д)»), записать значения по карте, сохранить в `out_path`. Числа — как числа (не строки), чтобы формулы Excel работали. Дату — строкой ДД.ММ.ГГГГ в C4. Прочерк «—» для несчитаемого.
- Форматирование/формулы шаблона НЕ ломать (openpyxl сохраняет их).

## Критерии приёмки
- `python tests/test_excel_smoke.py` заполняет оба листа за дату с данными (например 2026-06-10 — там есть ФАКТ) и сохраняет файлы в `output/`.
- В сохранённом файле: C3/C4 заполнены; D8/F8, D9/F9, D10/F10, D14/F14, D15/F15, D19/F19, D23 содержат значения; формульные ячейки (D16/F16/D17/F17/D18/F18, H*) НЕ перезаписаны (остались формулами).
- Числовые ячейки имеют числовой тип.
- Только чтение из 1С; запись только в выходной xlsx.
