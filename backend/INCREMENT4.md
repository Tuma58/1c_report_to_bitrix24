## Задача 1. Наценка на ЗЧ — формула «только запчасти»

Владелец выбрал 1-й вариант. Поменять ФАКТ наценки:

`Наценка на ЗЧ, % ФАКТ = (Выручка по заказ-нарядам − Себестоимость материалов) / Себестоимость материалов × 100`

- Использовать **только** статью выручки «Выручка по заказ-нарядам» (запчасти), НЕ включать «Работы по заказ-нарядам» (труд).
- Себестоимость — «Себестоимость материалов» (как сейчас).
- Если себестоимость = 0 → None (прочерк).
- Строка отчёта «Выручка за день» остаётся ПОЛНОЙ (запчасти + работы) — не менять.

Реализация:
- В `IncomeExpenseRepository` добавить выдачу выручки по статьям раздельно, например метод
  `revenue_breakdown(division_key, start, end) -> {"parts": .., "works": .., "cost": ..}`
  (статьи: parts = «Выручка по заказ-нарядам», works = «Работы по заказ-нарядам», cost = «Себестоимость материалов»).
- `revenue_and_cost` может остаться (total = parts+works, cost) или выражаться через breakdown.
- В `MetricsService._markup_pct_fact` считать по parts. Обновить комментарий: TODO(наценка-на-ЗЧ) → РЕШЕНО (parts-only по решению владельца), проверенные значения: Арсенал 49.2%, Реф 63.0% (01.07.2026).

Проверено на 01.07.2026: parts=187 717, cost=125 807 → 49.2%.

## Задача 2. Сводная таблица по всем организациям

Одна таблица: строка = организация, столбцы = показатели (ФАКТ; где есть план — можно добавить столбцы ПЛАН/%).

Организации (6 строк, из `REPORT_TO_DIVISION_DESCRIPTION`):
Арсенал, Реф. Сервис, Шаркер, ЦКР, Мойка на ул. Строителей, Мойка на ул. Ульяновская.

Столбцы (ФАКТ): Закрыто ЗН, Нормочасы, Выработка/мастер, Выручка, Себестоимость ЗЧ, Маржа, Маржинальность %, Средний чек, Наценка на ЗЧ %, Незакрытые ЗН >1дн.

⚠️ ВАЖНО (разные наборы показателей — не баг, а бизнес-реальность):
- Арсенал, Реф.Сервис — все столбцы валидны.
- Шаркер, ЦКР — длинные ремонты, дневных движений в регистре нет → выручка/себест/нормочасы часто 0. Это ожидаемо (их реальные ФАКТ идут из плана помесячно; отдельная форма — позже). В сводной показывать как есть (0 / прочерк).
- Мойки — выручка в статье «Работы» (услуги), запчастей/себестоимости нет → Наценка/Себест = прочерк; «Закрыто ЗН» = машин обслужено.
- Где значение неприменимо/None — выводить «—», не падать.

Реализация:
- `src/consolidated.py`: `ConsolidatedReporter` с методами `daily(day)` и `weekly(any_date)`, возвращает список строк (по орг.) и:
  - печатает таблицу в консоль (выравненную);
  - сохраняет xlsx: `output/Сводная_дневная_<date>.xlsx` / `output/Сводная_недельная_<week>.xlsx` — один лист, строки=орг., колонки=показатели, числа числовым типом. Заголовок с датой/неделей.
- Конфиг порядка организаций — список report_name.

## Задача 3. Недельные показатели

Добавить `MetricsService.weekly(report_name, any_date)` — период = ISO-неделя (Пн–Вс), содержащая any_date.

- ФАКТ: те же формулы, но интервал [понедельник, следующий понедельник):
  - Закрыто ЗН за неделю, нормочасы за неделю (Σ по закрытым ЗН недели), выработка/мастер = нормочасы_недели / уник. мастера недели, выручка/себест за неделю (регистр Period в неделе), наценка parts-only за неделю.
- ПЛАН недельный: месячное значение ÷ дней в месяце × 7 (раздел 3.3 ТЗ). Средний чек — как есть (не масштабировать).
- Реорганизация репозиториев: ввести интервал по диапазону. Добавить `_range_interval(field, start, end)` и period-варианты методов
  (`closed_orders_period`, `works`, `executors`, `revenue_breakdown`, `open_orders_older_than_1day` — по дате конца периода),
  а `daily()` выразить как период [day, day+1). Существующие результаты дневного расчёта НЕ должны измениться.
