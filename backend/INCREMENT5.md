# Инкремент 5 — заполнение НЕДЕЛЬНЫХ форм (Арсенал, Реф.Сервис)

Скоуп: заполнять листы «Арсенал (Н)» и «Реф.сервис (Н)» из `MetricsService.weekly()` + разбивку по дням из `daily()`. Метрики уже готовы (Инкремент 4) — задача про Excel-заполнение и разбивку по дням.

## Шаблон
- Тот же `backend/templates/report_template.xlsx`. Заполнять КОПИЮ, сохранять в `output/`.
- Листы: «Арсенал»→«Арсенал (Н)», «Реф. Сервис»→«Реф.сервис (Н)» (идентичный макет).

## Карта ячеек (листы (Н), одинаковые)
Значения ПЛАН — колонка **D**, ФАКТ — **F**.

| Ячейка | Показатель | Источник |
|---|---|---|
| C3 | АТЦ | report_name |
| C4 | Неделя | «ДД.ММ–ДД.ММ.ГГГГ» (week_start–week_end) |
| C5 | Руководитель | пока пусто «» (данных нет — пометить) |
| D9 / F9 | Нормочасы за неделю | weekly.normhours.plan / .fact |
| D10 / F10 | Ср. выработка мастера в день | weekly.output_per_master.plan / .fact ⚠ (см. прим.) |
| D11 / F11 | Закрыто ЗН за неделю | weekly.closed_orders.plan / .fact |
| D15 / F15 | Выручка за неделю | weekly.revenue.plan / .fact |
| D16 / F16 | Себестоимость ЗЧ | weekly.cost.plan / .fact |
| D20 / F20 | Наценка на ЗЧ (%) | weekly.markup_pct.plan / .fact |

**Раздел 3 «ВЫРУЧКА ПО ДНЯМ» — строки 24–30 (Пн..Вс):**
| Колонка | Что | Источник (daily() по дню) |
|---|---|---|
| D24..D30 | Выручка | day.revenue.fact |
| F24..F30 | Нормочасы | day.normhours.fact |
| H24..H30 | ЗН закрыто | day.closed_orders.fact |

Порядок: строка 24 = понедельник … строка 30 = воскресенье (week_start + смещение).

**НЕ ЗАПИСЫВАТЬ (формулы Excel):** D17/F17 (Маржа), D18/F18 (Маржинальность), D19/F19 (Средний чек), D20/F20? — нет, D20/F20 писать (наценка не формула). Колонка I раздела 3 (I24:I31 — Ср.чек), строка 31 ИТОГО (D31/F31/H31), вся колонка H разделов 1–2 (%). Их не трогать.
Объединённые: C3/C4/C5, писать в левую ячейку.

⚠️ Примечание (пометить на подтверждение владельца): строка 10 «Ср. выработка мастера в **день**», а `weekly.output_per_master` = нормочасы_недели / мастера (за неделю). Возможно, нужно ÷ рабочие дни. Пока класть weekly.output_per_master как есть, флаг в комментарии.

## Что сгенерировать / изменить
```
src/metrics.py         # week_daily_breakdown(report_name, any_date) -> list[DailyMetrics] (7 дней Пн..Вс)
src/excel_reporter.py  # fill_weekly(report_name, weekly_metrics, day_metrics(list[7]), out_path) + маппинг листа (Н)
tests/test_weekly_excel.py
```
- `week_daily_breakdown`: для недели any_date вернуть 7 `DailyMetrics` по дням Пн..Вс (переиспользовать daily()).
- `fill_weekly`: грузит копию шаблона, находит лист (Н), пишет по карте, сохраняет. Числа — числовым типом; «—» для None; дата-неделя строкой в C4.

## Критерии приёмки
- `tests/test_weekly_excel.py` заполняет «Арсенал (Н)» и «Реф.сервис (Н)» за неделю с планом (июль 2026), сохраняет в output/.
- В сохранённом файле: C3/C4 заполнены; D9/F9, D11/F11, D15/F15, D16/F16, D20/F20 — значения; раздел 3 (D24:H30) заполнен по дням; сумма D24:D30 (ИТОГО) ≈ weekly.revenue.fact.
- Формульные ячейки (D17/F17, D18/F18, D19/F19, I24:I31, D31/F31/H31, H-колонки) НЕ перезаписаны.
- Только GET из 1С; запись только в xlsx.
