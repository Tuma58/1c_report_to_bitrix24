# Инкремент 6 — формы МОЕК (дневная + недельная)

Направление «Мойка» = 2 локации: «Мойка на ул. Строителей» → 1С «Автомойка»; «Мойка на ул. Ульяновская» → 1С «Мойка Ульяновская». Показатели простые: машин обслужено, выручка, средний чек. Себестоимости/наценки/нормочасов нет.

## Wash-специфичный расчёт (важно — сервисная логика не подходит)
Добавить в `MetricsService`:
- `daily_wash(report_name, day) -> WashMetrics` и `weekly_wash(report_name, any_date) -> WashMetrics`.
- Показатели WashMetrics:
  - `cars` (Машин обслужено): ФАКТ = число закрытых ЗН за период (как closed_orders.fact); ПЛАН = план-показатель **код 000000074** «Машин обслужено» (÷ дней месяца для дня; ×7 для недели).
  - `revenue` (Выручка): ФАКТ = выручка из регистра (все статьи выручки, как сейчас — у моек это «Работы по заказ-нарядам»); ПЛАН = код **000000067** «Выручка за месяц» (÷ дней; ×7).
  - средний чек — НЕ считаем в коде (формула Excel).
- Переиспользовать репозитории (closed_orders_period, revenue_and_cost_period). Наценку/себест не трогаем.
- `plan_mapping` уже содержит «машин_обслужено»:000000074 и «выручка»:000000067.

Разбивка по дням недели (для динамики): `week_wash_daily_breakdown(report_name, any_date) -> list[WashMetrics]` (7 дней Пн..Вс).

## Карта ячеек — лист «Мойка (Д)» (ОДИН лист, ДВА блока)
Блок 1 — «Мойка на ул. Строителей» (division «Автомойка»):
| Ячейка (D=ПЛАН / F=ФАКТ) | Показатель |
|---|---|
| C3 | АТЦ = «Мойки» |
| C4 | Дата |
| D8 / F8 | Машин обслужено |
| D9 / F9 | Выручка за день |
Блок 2 — «Мойка на ул. Ульяновская» (division «Мойка Ульяновская»):
| D14 / F14 | Машин обслужено |
| D15 / F15 | Выручка за день |

НЕ ПИСАТЬ (формулы): D10/F10, D16/F16 (Средний чек), H-колонки (%).

## Карта ячеек — лист «Мойка (Н)» (ОДИН блок = СУММАРНО по обеим мойкам)
⚠️ ДОПУЩЕНИЕ: недельная форма — суммарно (Строителей + Ульяновская). Если владелец захочет пофилиально — переделаем на 2 файла.
Раздел 1 (D=ПЛАН, F=ФАКТ):
| D9 / F9 | Машин за неделю (сумма обеих) |
| D10 / F10 | Выручка за неделю (сумма обеих) |
НЕ ПИСАТЬ: D11/F11 (Средний чек — формула), H-колонки.

Раздел 2 «ДИНАМИКА ПО ДНЯМ» строки 15–21 (Пн..Вс):
| D15..D21 | Машин (сумма обеих за день) |
| F15..F21 | Выручка (сумма обеих за день) |
| I15..I21 | Операторов (уник. исполнителей за день, обе мойки) — приблизительно; если сложно, оставить пусто |
НЕ ПИСАТЬ: H15..H21 (Ср.чек формула), строка 22 ИТОГО (D22/F22/H22 = SUM/формулы).

Объединённые: C3/C4/C5 — писать в левую ячейку. C5 (Руководитель) — пусто.

## Критерии приёмки
- daily_wash('Мойка на ул. Строителей', 2026-07-01): машин>0, выручка>0 (сверить с ФАКТ 01.07: Автомойка 37 424 выручка).
- `fill_daily_wash` заполняет оба блока листа «Мойка (Д)», сохраняет в output/. Формулы Ср.чек (D10/D16) целы.
- `fill_weekly_wash` заполняет «Мойка (Н)»: раздел 1 суммарно, раздел 2 по дням; SUM(D15:D21)=машин недели, SUM(F15:F21)=выручка недели. Формулы (H, строка 22) целы.
- ПЛАН машин = код 074, выручка = 067 (÷дней; ×7 для недели). Где плана нет — «—».
- Только GET из 1С; запись только в xlsx.
