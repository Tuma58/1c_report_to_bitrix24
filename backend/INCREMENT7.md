## Расчёт — новый `ShopMetrics` + методы (Шаркер=«Шаркер Трейлер», ЦКР=«Кузовной»)
`MetricsService.daily_shop(report_name, day) -> ShopMetrics`. Поля:

### ВЫРАБОТКА
- `normhours`: ПЛАН = код 000000029 ÷ дней месяца; ФАКТ = дневной код 000000072 по `ДатаФиксации`.
- `output_per_master`: ПЛАН = код 000000068 ÷ дней; ФАКТ = дневной код 000000073 по `ДатаФиксации`.
- (только ЦКР) `output_cost` «Стоимость выработки»: **ПЛАН/ФАКТ = None (BLOCKED — формула не определена)**.

### ПАЙПЛАЙН (значение 1С на дату day; статусы через References.statuses())
Терминальные = {Закрыт, Отказ, Архив}. Все пайплайн-фильтры по `ПодразделениеКомпании_Key`.
- `active` Активных ЗН = НЕ терминальные И НЕ «Заявка». (проверено Шаркер: 40)
- `ready_today` Готовы к выдаче сегодня = НЕ терминальные И `ПлановаяДатаВыдачи` in [day, day+1).
- `awaiting_parts` Ожидают ЗЧ = статус in {«Ожидание автозапчастей клиента», «Ожидание деталей»}.
- `overdue` Просрочены = исключить `Закрыт` и `Отказ` (в коде также исключается терминальный `Архив`) И `ПлановаяДатаВыдачи gt datetime'1901-01-01T00:00:00' and lt day`. (⚠️ 1С OData НЕ поддерживает `ne null` — использовать нижнюю границу даты для отсева пустых). (проверено Шаркер: 114)
- `planned_close_week` Планируется закрыть на неделе = НЕ терминальные И `ПлановаяДатаВыдачи` in [понедельник, следующий понедельник) недели day.

### ФИНАНСЫ
- `closed_orders`: ФАКТ = число закрытых ЗН за день (closed_orders_period); ПЛАН = выручка_план/средний_чек (067/069) — как у сервисных.
- `revenue_closed` Выручка закрытых ЗН: ФАКТ = выручка из регистра за день (revenue_breakdown_period → parts+works, без НДС); ПЛАН = код 067 ÷ дней. (у длинных ремонтов дневной ФАКТ часто 0 — это норма).
- `payments` Поступления оплат: ФАКТ = см. ниже; ПЛАН = None.
- `insurance_count`, `insurance_sum` Страховые ЗН прошлых мес.: ФАКТ = закрытые с 30.12.2025 по дату отчёта включительно ЗН с видом ремонта «Страховой ремонт термофургонов» / «Кузовной страховой ремонт», у которых нет подчинённой счёт-фактуры или в регистре «Оригинал документа получен» по счёт-фактуре не заполнена дата «Получен КА» (`InformationRegister_олОригиналДокументаПолучен.Дата2`), и при этом есть неоплаченный остаток больше 1 рубля. Технически фильтр периода: `ДатаЗакрытия ge 30.12.2025` и `ДатаЗакрытия lt дата_отчёта + 1 день`. Оплаты берутся из `AccumulationRegister_ВзаиморасчетыКомпании_RecordType`: `Recorder_Type = Document_Выписка`, `Сделка_Type = Document_ЗаказНаряд`, `ВидОперации = ПогашениеДебиторскойЗадолженности`. Полностью оплаченные исключаются; частично оплаченные остаются с суммой остатка. База долга = максимум из суммы ЗН и максимальной суммы счёт-фактуры по этому ЗН, чтобы не задваивать повторные СФ. ПЛАН = 0 (в шаблоне).

## Поступления оплат — через договор→подразделение (новый репозиторий)
`PaymentRepository.payments_period(division_key, start, end) -> float`:
1. Статья ДДС «Оплата от покупателя» → Ref_Key (Catalog_СтатьиДДС, Description eq 'Оплата от покупателя'). Резолвить в References (метод `dds_article(name)`).
2. Договоры подразделения: `Catalog_ДоговорыВзаиморасчетов` where `Подразделение_Key eq guid'div'` → множество Ref_Key. (кэшировать)
3. Выписки: `Document_Выписка` where `СтатьяДДС_Key eq guid'оплата' and Date in [start,end)` → выбрать те, у кого `ДоговорВзаиморасчетов_Key` ∈ множестве договоров подразделения; сумма `СуммаДокументаПриход`.
   (оплат в день немного — фильтровать в Python по множеству договоров; НЕ строить огромный OR.)
- ⚠️ Не включать «Предоплата от покупателя» (отдельная статья) — только «Оплата от покупателя».

## Карты ячеек (ВНИМАНИЕ: у Шаркер и ЦКР РАЗНЫЕ номера строк)
Пайплайн-значение писать в колонку **D** (под заголовком «Значение (1С)», объединено D:F — писать в D).

### Лист «Шаркер (Д)»
- C3 АТЦ=«Шаркер», C4 Дата
- ВЫРАБОТКА: D8 нормочасы_план / F8 нормочасы_факт; D9 выработка_план / F9 выработка_факт
- ПАЙПЛАЙН (D): D13 active, D14 ready_today, D15 awaiting_parts, D16 overdue, D17 planned_close_week
- ФИНАНСЫ: D21/F21 closed_orders (план/факт), D22/F22 revenue_closed, F23 payments (D23 пусто/«—»), F24 «—» (D24=0 в шаблоне), F25 «—» (D25=0)
- НЕ ПИСАТЬ формулы: колонка H (%).

### Лист «ЦКР (Д)» (сдвиг из-за строки 10 «Стоимость выработки»)
- C3 АТЦ=«ЦКР», C4 Дата
- ВЫРАБОТКА: D8 нормочасы_план / F8 нормочасы_факт; D9 выработка_план / F9 выработка_факт; D10 «—» / F10 «—» (стоимость выработки BLOCKED)
- ПАЙПЛАЙН (D): D14 active, D15 ready_today, D16 awaiting_parts, D17 overdue, D18 planned_close_week
- ФИНАНСЫ: D22/F22 closed_orders, D23/F23 revenue_closed, F24 payments (D24 пусто), F25 «—» (D25=0), F26 «—» (D26=0)
- НЕ ПИСАТЬ: колонка H (%).
